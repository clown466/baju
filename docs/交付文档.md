# 短剧扒剧与仿写 —— 交付文档

> 版本：2026-07（master 分支 `514598e`）
> 仓库：git@github.com:clown466/baju.git
> 本文档面向接手者 / 需要与其他程序集成的开发者。

---

## 1. 程序简介

输入一部短剧的视频文件夹，程序自动完成五个阶段：

| 阶段 | 功能 | 使用的模型 |
|------|------|-----------|
| ① 扒剧 | 逐集分析视频，产出原剧剧本（台词+画面+分镜） | Gemini（视频多模态） |
| ② 拉片报告 | 汇总全剧剧本，生成结构化拉片分析报告 | 文本 LLM |
| ③ 新剧设定 | 基于报告给出仿写新剧的设定建议，可人工修改后润色 | 文本 LLM |
| ④ 分集大纲 | 生成新剧的分集大纲 | 文本 LLM |
| ⑤ 新剧剧本 | 按大纲逐集生成新剧剧本（支持批量+附加要求） | 文本 LLM |

每个阶段的产物都可以在网页上查看、编辑、保存，并可导出。

## 2. 技术栈与架构

```
浏览器（Vue 3 SPA, hash 路由）
   │  fetch /api/*   +  SSE /api/projects/{pid}/events（进度推送）
   ▼
FastAPI 后端（uvicorn, 127.0.0.1:8000）
   ├─ BatchRunner：并发批任务引擎（asyncio, 可取消, 带重试）
   ├─ GeminiVideo：阶段① 视频分析（google-genai SDK，双通道）
   ├─ TextLLM：阶段②-⑤ 文本生成（OpenAI 兼容 /chat/completions, httpx）
   ├─ ProjectStore：产物存文件系统（markdown/json）
   └─ StatusDB：分集状态存 SQLite
```

- **后端**：Python 3.12+（打包用 3.14）、FastAPI、uvicorn、pydantic v2、PyYAML、httpx、google-genai
- **前端**：Vue 3 + Vite 5 + vue-router 4（hash 模式），无 UI 库，纯 JS
- **测试**：后端 pytest（50 用例），前端 Vitest + @vue/test-utils（45 用例）

## 4. 配置（config.yaml）

位于 `backend/config.yaml`（打包模式下位于 exe 同级目录，首次运行自动生成默认文件）。
**推荐通过网页「⚙ 模型设置」页修改，保存即热生效，无需重启。**

```yaml
gemini:                # 阶段① 视频识别
  api_key: "..."
  model: "gemini-2.5-pro"
  base_url: null       # null=Google 官方；填中转站地址则走中转原生通道
  upload: "files"      # files=Files API（原生密钥，无大小限制）
                       # inline=内联上传（中转站可用，单集 ≤19MB）
text_llm:              # 阶段②-⑤ 文本生成（OpenAI 兼容接口）
  provider: "lemon"    # 当前使用的服务商名
  providers:
    lemon:
      base_url: "https://xxx/v1"
      api_key: "sk-..."
      model: "..."
concurrency: 3         # 阶段①/⑤ 批量并发数
retries: 2             # 失败自动重试次数
data_dir: "data"       # 数据目录
```

### 视频通道说明（重要）
- **Google 原生密钥**：`base_url: null` + `upload: files`，走 Files API，无单集大小限制。
- **中转站（new-api 类）**：`base_url: https://中转站域名` + `upload: inline`，模型名可能需要前缀（如 `[L]gemini-3.1-pro-preview`）。中转站不代理 Files API，所以视频以 base64 内联传输，**单集需 ≤19MB**，超出会返回中文错误提示。

## 5. 运行方式

### 开发模式（两个终端）
```bash
cd backend  && pip install -r requirements.txt && python -m app.main   # 127.0.0.1:8000
cd frontend && npm install && npm run dev                              # localhost:5173（/api 代理到 8000）
```
注意：V2rayN 等代理的 TUN 模式会拦截 localhost，需关闭 TUN 或设置绕过局域网。

### 生产/单进程模式
`npm run build` 后直接跑后端即可——后端检测到 `frontend/dist` 存在时会自动托管静态页面，访问 `http://127.0.0.1:8000` 即完整应用。

### 桌面单文件 exe
```bash
cd frontend && npm run build
cd ../backend && python -m PyInstaller --onefile --clean --name 短剧仿写助手 \
  --add-data "../frontend/dist;static" --collect-submodules uvicorn \
  --collect-all google.genai desktop.py
```
产物：`backend/dist/短剧仿写助手.exe`。双击运行自动开浏览器；`config.yaml` 与 `data/` 生成在 exe 同级目录。

### 测试
```bash
cd backend  && python -m pytest -q     # 50 passed
cd frontend && npm test -- --run       # 45 passed
```

## 6. REST API 一览（与其他程序集成的主要入口）

Base：`http://127.0.0.1:8000/api`，请求/响应均为 JSON（UTF-8），错误统一为 `{"detail": "中文错误信息"}` + 4xx/5xx。CORS 全开放。

### 设置
| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/settings` | 读取当前模型配置 |
| PUT | `/settings` | 更新配置 `{gemini, text_llm}`，写回 yaml 并热生效 |

### 项目
| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/projects` | 创建项目 `{name, video_dir}`，自动扫描分集（按文件名自然排序） |
| GET | `/projects` | 项目列表 |
| GET | `/projects/{pid}` | 项目详情（含各集状态 pending/running/done/failed） |
| PUT | `/projects/{pid}/episodes-mapping` | 修改 集数↔文件 映射 |

### 阶段① 扒剧
| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/projects/{pid}/stage1/start` | 批量开始 `{episodes: [1,2] 或 null=全部未完成}` |
| POST | `/projects/{pid}/stage1/cancel` | 取消批任务 |
| GET/PUT | `/projects/{pid}/episodes/{ep}/script` | 读/写单集原剧剧本 `{content}` |

### 阶段②-④ 产物
| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/projects/{pid}/stage2/generate` | 生成拉片报告 |
| POST | `/projects/{pid}/stage3/suggest` | 生成新剧设定建议 |
| POST | `/projects/{pid}/stage3/refine` | 按人工修改稿润色 `{draft}` |
| POST | `/projects/{pid}/stage4/generate` | 生成分集大纲 |
| GET/PUT | `/projects/{pid}/artifacts/{kind}` | 读/写产物，kind ∈ analysis / settings / outline |

### 阶段⑤ 新剧剧本
| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/projects/{pid}/stage5/generate` | 单集生成 `{episode, extra}`（extra=附加要求） |
| POST | `/projects/{pid}/stage5/start` | 批量生成 `{episodes, extra}` |
| GET/PUT | `/projects/{pid}/scripts/{ep}` | 读/写单集新剧剧本 |

### 导出与进度
| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/projects/{pid}/export?which=original\|new` | 导出全部剧本（合并为单个纯文本，text/plain） |
| GET | `/projects/{pid}/events` | **SSE** 实时进度流，事件 data 为 JSON（含 episode/status 等），批任务期间推送 |

## 7. 数据存储布局（文件级集成入口）

所有产物是纯文本文件，另一个程序可直接读写 `data/` 目录：

```
data/
├─ status.db                        # SQLite：分集状态(pid, ep, status, error)
└─ projects/{pid}/
   ├─ project.json                  # {id, name, video_dir, episodes:[{episode,file}]}
   ├─ episodes/ep001.script.md      # 阶段① 原剧剧本（每集一个）
   ├─ analysis/report.md            # 阶段② 拉片报告
   ├─ settings/new_drama.md         # 阶段③ 新剧设定
   ├─ outline/outline.md            # 阶段④ 分集大纲
   └─ scripts/ep001.md              # 阶段⑤ 新剧剧本（每集一个）
```

支持的视频格式：mp4 / mkv / mov / avi / ts / webm / flv。

## 8. 与另一个程序结合的建议方式

1. **REST 集成（推荐）**：对方程序直接调用第 6 节的 API。全流程编排示例：
   `POST /projects` → `POST stage1/start` → 订阅 SSE 等全部 done → `stage2/generate` → `stage3/suggest`（或直接 PUT settings 产物跳过人工环节）→ `stage4/generate` → `stage5/start` → `GET /export?which=new`。
2. **文件级集成**：共享 `data_dir`，对方程序直接读取/投喂 markdown 产物（注意：分集状态在 status.db，直接写文件不会更新状态）。
3. **代码级集成**：`from app.main import create_app` 将本应用作为子应用挂载到对方 FastAPI（`app.mount("/baju", ...)`），或复用 `app/llm.py`、`app/engine.py` 模块。
4. 端口 8000 写死在 `app/main.py: run()`，若与对方程序冲突，改该处或用 `uvicorn.run(create_app(load_config()), port=xxxx)` 自行启动。

## 9. 已知边界与注意事项

- 服务只绑定 127.0.0.1，**无鉴权**，不要直接暴露公网。
- 中转站内联通道单集视频 ≤19MB；超大视频请压缩或换 Google 原生密钥。
- 阶段②依赖阶段①至少一集 done；③依赖②；④依赖③；⑤依赖④（缺前置产物返回 400 中文提示）。
- `config.yaml` 含密钥，已在 .gitignore 中，勿提交。
- Windows + 代理 TUN 模式会导致 localhost 拒绝连接（见 5 节）。
