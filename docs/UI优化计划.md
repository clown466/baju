# 前端 UI 优化计划 —— 深色影视工具风 + 交互体验全升级

> 日期：2026-07-06　方向已确认：深色影视工具风／视觉+交互全升级／不引入组件库（手写 CSS + 设计令牌）
> 约束：现有 45 个前端测试必须保持通过；新交互组件按 TDD 补测试；不改后端。

## 0. 现状问题清单

**视觉**（style.css 仅 19 行）：
- 原生无样式控件，白底默认字体，无色彩/间距/圆角/阴影体系
- 表格 1px 灰边框；tabs 仅一条下划线；顶栏是唯一"有设计"的元素
- 无 favicon、无页面过渡、滚动条系统默认

**交互体验**：
- 批量任务只有文字状态，无进度条（如 19/20 无可视化）
- 操作成功无反馈（保存/设置只有一行小字或什么都没有）
- 错误只显示一行红字，位置分散且不会自动消失
- 无加载骨架屏（只有"加载中…"）；无空状态引导
- "批量生成全部""重生成"等覆盖性/耗时操作无二次确认
- EditorPane 编辑无未保存提示，切换即丢失
- 表格无 hover、状态无图标、按钮无 loading spinner

## 1. 设计方向：深色影视工作台

- **底色层次**：`#0d1117`（页面）→ `#161b22`(卡片) → `#1f2733`(顶栏/悬浮)
- **强调色**：琥珀金 `#e8a33d`（影视/片场灯光感），hover 加亮；辅助色沿用语义色系
- **语义色**：成功 `#3fb950`、失败 `#f85149`、进行中 `#d29922`、信息 `#58a6ff`
- **文字**：主 `#e6edf3`、次 `#8b949e`、禁用 `#484f58`
- **字体**：中文界面 system-ui/微软雅黑；剧本正文与编辑区用衬线或等宽混排提升"剧本感"
- **形态**：8px 间距栅格、6px 圆角、卡片弱阴影+1px 边框、深色滚动条

## 2. 实施阶段

### Phase A：设计令牌 + 全局换肤（纯 CSS，零逻辑改动）
1. `src/style.css` 重写：`:root` CSS 变量（色板/间距/圆角/阴影/字号），body 深色底
2. 全局元素样式：button（主/次/危险三态 + hover/active/disabled）、input/textarea/select（深色描边、focus 光圈）、table（去竖线、行 hover、斑马纹）、a、h1/h2、滚动条
3. 顶栏升级：logo 标题 + 右侧「模型设置」入口（从 ProjectList 移到 App.vue 顶栏，全局可达）
4. tabs 重做：胶囊式页签 + 底部指示条 + 完成态小圆点
5. `index.html`：favicon（🎬 emoji svg）、深色 `color-scheme`
- **验收**：所有页面视觉统一深色；45 测试全绿（此阶段不动模板结构，仅 App.vue 顶栏微调 + 对应测试更新）

### Phase B：组件级视觉重构（改模板 class，不改行为）
1. ProjectList：项目改卡片网格（名称/集数/封面占位），新建表单收进卡片
2. ProjectDetail：五阶段 tabs 显示流程感（①→⑤ 步骤条式，已完成阶段打勾）
3. Stage1/Stage5 表格：状态列改「彩色徽章 + 图标」（done ✓ 绿 / failed ✕ 红 / running 转圈黄）
4. Settings：分组卡片化（已有基础，统一到令牌体系），密钥输入框加显隐切换眼睛按钮
5. EditorPane：工具栏贴顶吸附，编辑区等宽字体、行号感背景，查看/编辑切换动画
- **验收**：逐页对照检查；受影响测试同步更新选择器；45+ 全绿

### Phase D：细节打磨与收尾
1. 页面/页签切换 150ms 淡入过渡；状态徽章变色过渡
2. 响应式：≤768px 时表格可横滚、卡片单列（窗口缩放不破版）
3. `npm run build` 重新打包 exe 验证（深色页面在打包模式同样生效）
4. 全量回归：前端测试 + 后端 50 测试 + 手工冒烟
5. 提交推送（每个 Phase 一个 commit）

## 3. 不做什么（明确排除）

- 不引入 UI 组件库/图标库（图标用内联 SVG 或 emoji）
- 不做浅色主题与主题切换（已选定深色单主题）
- 不改路由结构、API 层、后端任何代码
- 不做国际化、无障碍深度优化（保留基本语义标签）

## 4. 风险与对策

| 风险 | 对策 |
|---|---|
| 改模板导致现有测试大面积失败 | Phase A 只动 CSS；B/C 每改一个组件立刻跑对应测试文件 |
| Toast/Confirm 替换 error 文案破坏"保存失败展示后端错误"等既有断言 | 保留 `.error` class 渲染于 toast 内，或同步更新断言 |
| 深色下 Markdown 预览可读性差 | `.markdown` 单独调优（标题层级色、代码块底色、引用条） |
| exe 打包忘更新前端 | Phase D 固定包含 rebuild + 冒烟 |

## 5. 执行顺序与提交计划

```
A 令牌换肤   → commit: style: 深色影视工具风全局换肤（设计令牌体系）
B 组件视觉   → commit: style: 组件级视觉重构（卡片/步骤条/状态徽章）
C 交互体验   → commit: feat: Toast/进度条/二次确认/骨架屏等交互体验升级
D 打磨收尾   → commit: chore: UI 细节打磨、响应式与 exe 重打包
```
